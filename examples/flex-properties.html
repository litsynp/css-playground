flex vs flexbox

flex and flexbox are the same thing.

display: flex

When we set display: flex on the container, its child element has default flex properties of flex: 0 1 auto. This means flex-grow: 0, flex-shrink: 1, and flex-basis: auto.

flex: 1

flex: 1 is shorthand for flex: 1 1 0, which means flex-grow: 1, flex-shrink: 1, and flex-basis: 0%.

  • flex: 1 1 0 means the element can grow and shrink as much as it can, and it starts at a size of 0.
  • flex 1 1 auto means the element can grow and shrink as much as it can, and it starts at its default size, which is determined by width attributes or parent > child size designation.
  • flex-basis: 0 and flex-basis: 0% are different in that 0% is relative to the flex container's inner main size. That means if the parent container doesn't have any explicit size(since you are using min-height), we cannot resolve the percentage value and it will fail to auto. Setting 0 on the other hand means setting the element size to 0. (source)

flex-basis

flex-basis sets the initital main size of the flex item. It sets the size of the content box unless otherwise set with box-sizing.

flex-grow

flex-grow specifies how much the item will grow relative to the rest of the flexible items inside the same container.

flex-shrink

flex-shrink specifies how much the item will shrink relative to the rest of the flexible items inside the same container.

  • Default value of flex-shrink should be 1.
  • The bigger the flex-shrink value, the more the item will shrink relative to the rest of the flexible items.
  • flex-shrink often requires the parent container to be a single-line flex container by setting flex-wrap: nowrap. Otherwise, the items will wrap to the next line instead of shrinking. (source)
  • flex-shrink is often used alongside flex-grow and flex-basis.
